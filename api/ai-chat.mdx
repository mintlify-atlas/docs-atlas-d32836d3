## Chat with AI assistant

<Card title="POST /v1/ai/chat" icon="messages">
  Send messages to the AI assistant with trip and page context.
</Card>

### Request

**Authentication:** Required (JWT token)

**Headers:**
- `Content-Type`: application/json
- `Authorization`: Bearer `<token>`

**Body parameters:**

<ParamField body="tripId" type="string" required>
  Trip ID the conversation belongs to
</ParamField>

<ParamField body="pageKey" type="string" required>
  Current page context (e.g., `"flights"`, `"transit"`, `"finance"`, `"itinerary"`, `"overview"`)
</ParamField>

<ParamField body="pageContext" type="object">
  Additional page-specific context data
</ParamField>

<ParamField body="messages" type="array" required>
  Array of chat messages
  <Expandable title="ChatMessage object">
    <ParamField body="role" type="string" required>
      Message role: `"user"` or `"assistant"`
    </ParamField>
    <ParamField body="content" type="string" required>
      Message text content
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="refresh" type="boolean" default="false">
  Whether to fetch real-time context (flight status, transit options, etc.)
</ParamField>

### Response

<ResponseField name="ok" type="boolean" required>
  Request success status
</ResponseField>

<ResponseField name="data" type="object">
  Chat response data
  <Expandable title="ChatResponse object">
    <ResponseField name="conversationId" type="string">
      Unique conversation ID
    </ResponseField>
    <ResponseField name="answer" type="string">
      AI assistant response text
    </ResponseField>
    <ResponseField name="highlights" type="array">
      Array of highlight strings
    </ResponseField>
    <ResponseField name="suggestedActions" type="array">
      Array of suggested next actions
    </ResponseField>
    <ResponseField name="sources" type="array">
      Data sources used for response
      <Expandable title="Source object">
        <ResponseField name="name" type="string">
          Source name (e.g., `"next_flight_status"`, `"trip_db_context"`)
        </ResponseField>
        <ResponseField name="status" type="string">
          Source status: `"ok"`, `"error"`, `"skipped_missing_inputs"`
        </ResponseField>
        <ResponseField name="fetchedAt" type="string">
          ISO 8601 timestamp
        </ResponseField>
        <ResponseField name="detail" type="string">
          Additional details or error message
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="degraded" type="boolean">
      Whether response is degraded due to unavailable real-time data
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="error" type="string">
  Error message if `ok` is false
</ResponseField>

### Example request

```bash
curl -X POST http://localhost:8080/v1/ai/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -d '{
    "tripId": "trip_abc123",
    "pageKey": "flights",
    "pageContext": {},
    "messages": [
      {
        "role": "user",
        "content": "What is the status of flight AA123 on 2026-03-15?"
      }
    ],
    "refresh": true
  }'
```

```bash
curl -X POST http://localhost:8080/v1/ai/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "tripId": "trip_abc123",
    "pageKey": "transit",
    "pageContext": {},
    "messages": [
      {
        "role": "user",
        "content": "How do I get from Times Square to Brooklyn Bridge?"
      }
    ],
    "refresh": true
  }'
```

### Page-aware context

The assistant fetches real-time data based on `pageKey` when `refresh` is `true`:

- **`flights`**: Extracts flight number and date from message, calls `/api/flights/status`
- **`transit`**: Extracts origin/destination from message ("from X to Y" format), calls `/api/transit/suggest`
- **`finance`**: Uses trip database for expense totals and budget data
- **`itinerary`**: Uses trip database for daily activities and timeline
- **`overview`**: Generic trip context with destination and dates

### Error responses

<ResponseField name="ok" type="boolean">
  `false` when error occurs
</ResponseField>

<ResponseField name="error" type="string">
  Error description
</ResponseField>

**Status codes:**
- `400` - Invalid input (missing tripId, pageKey, or messages)
- `403` - Unauthorized trip access
- `500` - Internal server error
